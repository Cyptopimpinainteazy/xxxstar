# GPU Node Setup (Linux)  

This doc shows how to prepare a Linux host to run `swarm-node` with real GPUs (NVIDIA or AMD). It covers drivers, Docker with GPU runtime, quick validation, and how to run the node binary.

## Summary
- Supported GPU stacks: NVIDIA (CUDA), AMD (ROCm), Vulkan/OpenCL backends
- Recommended: use Docker images with GPU access for easier deployment

## NVIDIA (CUDA) Quick Install
1. Verify GPU and driver support:
   - lspci | grep -i nvidia
   - sudo nvidia-smi

2. Install NVIDIA driver (example for Ubuntu):
   - sudo apt update && sudo apt install -y nvidia-driver-530
   - Reboot

3. Install CUDA toolkit (optional for development):
   - Follow NVIDIA CUDA install guide for your distribution

4. Install nvidia-container-toolkit for Docker GPU support:
   - distribution-specific instructions: https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html
   - After install: sudo systemctl restart docker
   - Test: docker run --rm --gpus all nvidia/cuda:12.1.1-base-ubuntu22.04 nvidia-smi

## AMD (ROCm) Quick Install
- Follow official ROCm install docs for your distribution (kernel and driver compatibility matters).
- Test with ROCm containers or `rocminfo` / `clinfo`.

## Vulkan / OpenCL
- For non-NVIDIA devices, ensure `vulkan-tools` or `clinfo` are installed and validate via `vulkaninfo` or `clinfo`.

## Run the Node (Quickstart)
1. Download the packaged release tarball and extract:
   - tar xzf gpu-swarm-linux-x86_64-v0.1.0.tar.gz
2. Copy binaries to `/usr/local/bin` and configs to `/etc/gpu-swarm/` (or run in place for testing):
   - sudo mkdir -p /etc/gpu-swarm && sudo cp node-config.toml /etc/gpu-swarm/
   - sudo cp swarm-node /usr/local/bin/
3. Environment variables (example):
   - export RUST_LOG=info
   - export GPU_BACKEND=cuda   # or `vulkan`, `opencl`
4. Run node:
   - sudo /usr/local/bin/swarm-node --config /etc/gpu-swarm/node-config.toml

5. Local admin GUI: a lightweight local web UI is available on `http://127.0.0.1:9101`.
   - It allows toggling participation (on/off), selecting GPU level (`low|medium|high`), scheduling on/off times, and shows live **Rewards**, **Uptime**, and **Score**.
   - The UI polls `/api/state` (JSON) and writes settings through `POST /api/state`.

If you run the Docker image, expose port 9101 to access the GUI from your host (e.g., `-p 9101:9101`).
## Docker (Recommended)
- Build a Docker image that includes the node binary and runtime deps.
- Example runtime tests with NVIDIA: docker run --rm --gpus all myrepo/gpu-swarm:latest /usr/local/bin/swarm-node --config /etc/gpu-swarm/node-config.toml

## Common Troubleshooting
- `nvidia-smi` not found → driver not installed or kernel mismatch
- Docker + GPU: ensure `nvidia-container-toolkit` installed and `docker info` shows `Runtimes: nvidia`
- Permission issues: run with a dedicated `swarm` user or use systemd unit file (examples included in release)

## Security Notes
- Coordinator endpoints accept task submissions; protect with firewall rules or run behind API gateway
- Use system user isolation and limit access to the config directory

## Local Admin Token & Registration
- On first run the node creates a local token at `$XDG_CONFIG_HOME/gpu-swarm/admin.token` (or `~/.config/gpu-swarm/admin.token`). Use it to login to the local UI or call `/api/login`.
- Use the `Register` button in the UI to create a 12-word recovery phrase and (prototype) wallet address. Save the phrase securely; the wallet address will be stored in `node-config.toml`.
- **Prototype note:** the current mnemonic is a prototype generator (non-BIP39). For production we will migrate to a standard BIP39 implementation and wallet encryption.
