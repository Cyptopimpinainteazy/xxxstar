# X3 Security Swarm Charter

This charter defines the allowed scope of the X3 Security Swarm. The swarm exists to preserve protocol integrity, maintain availability, reduce blast radius during incidents, and produce evidence that can survive audits, governance review, and disputes.

Automated defense is permitted for reversible containment only. The swarm may slow, isolate, pause, or quarantine protocol surfaces that satisfy published thresholds and quorum rules. Permanent sanctions such as long-lived blacklists, bond seizures, governance exclusions, or partner-chain threat propagation require a ratified follow-up process.

Due process applies to permanent consequences. Evidence hashes must be published, challenge windows must be time-bounded, and the affected actor must have a documented path to contest attribution. Emergency containment does not require prior consent; permanent punishment does.

Agent accountability is non-optional. Every agent class must be killable, every action must be auditable, and every configuration change must be attributable to a governance or operator event. The swarm has no immunity because it is automated.

The charter explicitly forbids counter-hacking, device interference, data destruction on third-party systems, off-chain punishment, or impersonation. The security model ends at X3-controlled infrastructure and protocol policy.
