Execution Flow
FOUR PHASES.
ZERO TRUST.
01
SETUP — Generate Secret

Compute secret S and hash H = sha256(S). Pre-fund inventory vaults on each chain. Negotiate amounts, assets, and timelocks. Set T_S (Solana) and T_E = T_S + Δ (Ethereum) for staggered safety windows.

S = random_bytes(32)
H = sha256(S)
// staggered timelocks
T_S = now + 3600
T_E = now + 7200
// fund vaults
vault.lock(USDC, 100_000)
02
LOCK — Deploy HTLCs

Call createHTLC on both chains with the same hash H. Funds lock in smart contracts. Wait for confirmations (5 on SVM, 12 on EVM) to guard against reorgs before proceeding.

// Solana HTLC
svm.createHTLC(
  recipient, H, T_S,
  USDC, 100_000
)
// Ethereum HTLC
evm.createHTLC(
  recipient, H, T_E,
  USDC, 99_950
)
03
FLASH — Execute Legs

Both chains fire flashloan→swap→repay in a single transaction each. If either reverts, the whole operation aborts. No secret is revealed. Funds stay locked in HTLC, pending timeout refund.

// Kamino (Solana, 0% fee)
kamino.flashBorrow(USDC, 1M)
jupiter.swap(USDC → SOL)
kamino.flashRepay(USDC, 1M)
// Balancer (ETH, 0% fee)
balancer.flashLoan(USDC, 1M)
uniswapV3.swap(USDC → ETH)
balancer.repay(USDC, 1M)
04
CLAIM — Reveal & Collect

Both legs succeed. Initiator calls claimHTLC(S) on Solana, revealing S on-chain. The relayer picks up the event and immediately calls claimHTLC(S) on Ethereum. Profit collected on both chains.

// reveal on SVM
svm.claimHTLC(id_S, S)
// emit: HtlcClaimed(S)
// relayer picks up S...
evm.claimHTLC(id_E, S)
// ✓ funds collected
// ✓ profit secured
// ✓ pools repaid
