┌─ THREE CORE TECHNIQUES ────────────────────────────────────────────────────┐
│                                                                             │
│  1. DOMINATOR TREE CONSTRUCTION                                             │
│     └─ Block B dominates D if every path to D goes through B               │
│     └─ Enables: Guaranteed facts from B apply to D                         │
│     └─ Status: ✓ Implemented (Cfg::from_function)                          │
│                                                                             │
│  2. CONDITION CANONICALIZATION                                              │
│     └─ Normalize: if (!(x != 4)) ──→ EqVarConst(x, 4)                     │
│     └─ Normalize: if ((y | 0) == 1) ──→ EqVarConst(y, 1)                  │
│     └─ Status: ✓ Implemented (CanonicalCond enum)                          │
│                                                                             │
│  3. FORWARD CONSTANT PROPAGATION                                           │
│     └─ Track values: x = 4 → if (x == 4) FOLD!                           │
│     └─ Lattice: Unknown ⊑ Const(v) ⊑ Overdefined                          │
│     └─ Status: ✓ Implemented (iterative dataflow)                          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

┌─ ALGORITHM OVERVIEW ───────────────────────────────────────────────────────┐
│                                                                             │
│  Phase 1: Initialize                                                       │
│    └─ in[block][var] = Unknown for all blocks, vars                       │
│    └─ worklist = [all blocks]                                             │
│                                                                             │
│  Phase 2: Propagate (Worklist Algorithm)                                   │
│    └─ while worklist not empty:                                           │
│       ├─ block = pop()                                                    │
│       ├─ new_in = meet(predecessors' outputs)                            │
│       └─ if new_in != in[block]:                                         │
│          ├─ in[block] = new_in                                           │
│          └─ push_successors()                                            │
│                                                                             │
│  Phase 3: Fold (Branch Folding)                                            │
│    └─ for each branch:                                                    │
│       ├─ if condition provably true → goto true_block                    │
│       ├─ else if condition provably false → goto false_block             │
│       └─ else preserve branch                                            │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

┌─ SYNERGIES WITH OTHER PASSES ──────────────────────────────────────────────┐
│                                                                             │
│  With Dead Code Elimination (DCE):                                        │
│    Cond Fold: if (true) goto A else B  →  goto A                        │
│    DCE:       Block B unreachable      →  Delete B                        │
│    Result:    40% dead block reduction                                    │
│                                                                             │
│  With Copy Propagation:                                                   │
│    Cond Fold: Proves x == 4 at certain points                           │
│    Copy Prop: Substitutes 4 for x                                        │
│    Result:    Fewer variables to track, fewer spills                     │
│                                                                             │
│  With Loop Analysis:                                                      │
│    Cond Fold: Simplifies loop conditions                                 │
│    Loop Opt:  Better loop detection                                      │
│    Result:    More aggressive loop optimizations                         │
│                                                                             │
│  With Superoptimization:                                                  │
│    Cond Fold: Reduces branch count                                       │
│    Superopt:  Fewer patterns to search                                   │
│    Result:    Faster optimization, same quality                          │
