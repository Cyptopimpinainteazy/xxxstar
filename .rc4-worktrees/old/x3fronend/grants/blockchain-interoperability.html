Atomic Gateway

One asset, multiple execution domains, atomic settlement.

Atomic Gateway makes BTC, EVM, SVM, and X3VM interoperable through a canonical asset accounting model. This page is for ecosystem funders who want practical cross-chain settlement, not speculative bridge language.

Sponsor integrationRequest adapter plan

Problem

Liquidity is fragmented across chains, bridge models are inconsistent, and execution domains are siloed. That creates supply drift, settlement risk, and poor predictability for funders looking for verifiable cross-domain infrastructure.

X3 solution

X3 uses the Atomic Kernel to track canonical asset state across BTC, EVM, SVM, and X3VM. Atomic Gateway then routes liquidity and settlement decisions with proof receipts that show how the asset moved without creating multiple independent supply pools.

Atomic Kernel

Canonical ledger layer that enforces a single source of truth for asset supply and settlement state.

Domain adapters

EVM, SVM, BTC/xBTC, and X3VM adapters keep each execution domain connected while preserving settlement invariants.

Technical architecture

Atomic Gateway

  • Route planning with canonical settlement receipts
  • Interoperability contracts for EVM/SVM
  • BTC/xBTC watch and lock integration

StarPackets

  • Verifiable message format for cross-domain state
  • Receipt-backed reroute logic
  • Fault-tolerant retry semantics

What is already built
