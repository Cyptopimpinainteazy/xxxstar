# Examples Directory

This directory contains code examples and patterns that the AI coding assistant should follow when implementing new features.

## What to Include

### Code Structure Patterns
- How you organize modules
- Import conventions
- Class/function patterns

### Component Patterns
- React component structure
- Prop handling
- State management

### Styling Patterns
- Tailwind CSS usage
- Custom CSS conventions
- Responsive design patterns

### Testing Patterns
- Test file structure
- Mocking approaches
- Assertion styles

### Integration Patterns
- API client implementations
- Data fetching patterns
- Error handling flows

## Example Structure
```
examples/
├── README.md              # This file - explains what each example demonstrates
├── components/            # React component patterns
│   └── Example.jsx       # Example component with best practices
├── hooks/                 # Custom React hook patterns
│   └── useExample.js     # Example hook
├── services/              # API service patterns
│   └── api-example.js    # Example API client
└── tests/                 # Testing patterns
    └── test-example.js   # Example test file
```

## Best Practices
1. More examples = better implementations
2. Show both what to do AND what not to do
3. Include error handling patterns
4. Document any gotchas or common pitfalls
