Email Reputation

Set funding outreach up for real deliverability.

This page explains how the funding system uses proper SMTP reputation, sender policy, and relay fallback to protect outreach to sponsors and reviewers.

Reputation-first email architecture

  • Set SPF records for the funding domain and any sending subdomains.
  • Publish DKIM keys and sign every outgoing message.
  • Use DMARC with a reporting policy to monitor abuse and delivery issues.
  • Route transactional messages through a trusted relay provider when self-hosted delivery fails.

Hybrid delivery model

Self-hosted first

Use your own mail server for direct control of funding outreach, with private IP reputation and domain-based records.

Relay fallback

Send through a transactional provider like Mailgun, SendGrid, or Postmark when the direct path is unavailable or to preserve deliverability volume.

Operational checklist

  • Implement SPF, DKIM, DMARC for the funding domain.
  • Publish BIMI and brand indicators if available.
  • Build a sending cadence that avoids bulk-sender reputation risk.
  • Monitor bounces, spam complaints, and inbox placement.
