# Examples Directory

This directory contains code examples and patterns that the AI coding assistant should follow when implementing new features.

## What to Include

### Code Structure Patterns
- How you organize modules
- Import conventions
- TypeScript type patterns

### Component Patterns
- React component structure (Server vs Client components)
- Prop handling with TypeScript
- State management patterns

### Styling Patterns
- Tailwind CSS usage
- CSS modules conventions
- Responsive design patterns

### Testing Patterns
- Test file structure
- Mocking approaches
- Assertion styles

### Integration Patterns
- Web3 wallet connection
- API client implementations
- Error handling flows

## Example Structure
```
examples/
├── README.md              # This file - explains what each example demonstrates
├── components/            # React component patterns
│   └── Example.tsx       # Example component with best practices
├── hooks/                 # Custom React hook patterns
│   └── useExample.ts     # Example hook
├── lib/                   # Utility function patterns
│   └── api-example.ts    # Example API client
└── tests/                 # Testing patterns
    └── test-example.tsx  # Example test file
```

## Best Practices
1. More examples = better implementations
2. Show both what to do AND what not to do
3. Include error handling patterns
4. Document any gotchas or common pitfalls
5. Use TypeScript throughout
