# Companion Agent Configuration

## Mission
After primary task completion, invoke the workspace post-task documentation prompt and update all related markdown files with evidence-backed changes.

## Invocation Contract
When the user says any of the following, run this flow:
- "resume"
- "post-task docs"
- "sync docs"
- "update related docs"
- "close out docs"

## Execution Flow
1. Read `.prompt.md`.
2. Gather evidence from recent commands and task outputs.
3. Locate related markdown files.
4. Apply updates to status, QA, readiness, and index docs.
5. Generate a session summary entry using the required output format.
6. Report completion with updated file list.

## Required Evidence Fields
- Task name
- Test results
- Build logs
- Timestamp
- Metrics

If any required field is missing, mark the result as PARTIAL and list missing evidence.

## Safety and Accuracy
- Never mark a phase complete without test/build evidence.
- Never invent metrics.
- Keep claims aligned with the actual repository state.
