x3-vm / architecture-overview

X3-VM
CROSS-CHAIN ATOMIC
COORDINATOR

A trust-minimized atomic swap system coordinating Ethereum (EVM), Solana (SVM), and X3 Chain (WASM) through Hash Time-Locked Contracts, flashloan execution, and X3-kernel's ComitV2 cross-VM dispatch mechanism.

X3 Finality
200ms
Flash finality target
Chains
3
EVM + SVM + X3VM
Partial Fills
ZERO
Atomic or full refund
ℹ
Design principle: The initiator funds two HTLCs with a common hash H=sha256(S). Flashloan legs execute atomically per-chain. Revealing S on one chain triggers claim on the other via relayer. If any leg fails — S is never revealed — both chains refund after their respective timelocks.

COORDINATOR PSEUDOCODEX3-LANG
// Phase 0: Setup
let S = random_bytes(32);
let H = sha256(S);
let T_S = now() + 3600;   // Solana: 1 hour
let T_E = now() + 7200;   // Ethereum: 2 hours

// Phase 1: Lock HTLCs
let id_S = svm.createHTLC(recipient, H, T_S, USDC, 100_000);
let id_E = evm.createHTLC(recipient, H, T_E, USDC, 99_950);
wait_confirmations(svm_id, 5);
wait_confirmations(evm_id, 12);

// Phase 2: Execute flash legs (atomic per-chain)
let svm_ok = try { svm.kamino_flash_arb(USDC, 1_000_000) };
let evm_ok = try { evm.balancer_flash_arb(USDC, 1_000_000) };

// Phase 3: Claim or Abort
if svm_ok && evm_ok {
  svm.claimHTLC(id_S, S);      // reveals S on-chain
  relayer.relay_claim(id_E, S); // claims EVM side
} else {
  // Wait for timeouts — both HTLCs auto-refund
  svm.refundHTLC(id_S); // after T_S
  evm.refundHTLC(id_E); // after T_E
}

x3-vm / security / attack-vectors

ATTACK
VECTORS

Exhaustive threat model analysis with mitigation status.

⚠
MEV / Front-running: Secret S is kept off-chain until both flashlegs confirm success. Private relay (Flashbots) and X3 encrypted mempool prevent transaction ordering attacks. Verify sha256(secret) == hash_lock on-chain.
✓
Partial Fill Prevention: Atomic executor guarantees either full cross-chain execution or complete abort. No partial state is possible. If EVM leg reverts, SVM leg's funds remain locked in HTLC until T_S timeout refund.
✓
Oracle Manipulation: System uses AMM pool prices (TWAP) exclusively. Trade sizes limited to <5% of pool depth. No reliance on centralized price feeds for execution logic.
!
X3-VM Runtime Risk: As an emerging WASM runtime, x3vm bugs could be catastrophic. Mitigation: multi-sig recovery, contract kill-switch, GPU verification layer, and formal spec validation. Monitor x3-kernel verifiers for payload/proof mismatches.

Relay : evm.claimHTLC(S) within relay window
Expected : Initiator: +10 USDC profit total. Pools: fully repaid. HTLCs: claimed.
TEST_002 // WRONG_PREIMAGE_CLAIM● PASS
Action : claimHTLC(id, 0xdeadbeef_WRONG)
Expected: REVERT: InvalidSecret — sha256(wrong) ≠ hash_lock
State : HTLC unchanged. Funds remain locked.
TEST_003 // EARLY_REFUND_BLOCKED● PASS
Action : refundHTLC(id) at t < T_S
Expected: REVERT: TooEarly — block_time() < timelock
After T_S: refundHTLC(id) at t ≥ T_S → PASS, funds returned to sender
TEST_004 // ABORT_ON_FAILED_LEG● PASS
Scenario : EVM flash succeeds, SVM flash reverts (insufficient liquidity)
Expected : No secret revealed. SVM TX reverts completely (atomic). EVM TX reverts.
Recovery : After T_S: refundHTLC on SVM. After T_E: refundHTLC on EVM.
Net loss : Gas fees only. Zero principal loss.
TEST_005 // FLASHLOAN_ATOMICITY● PASS
Setup : Borrow 1000 USDC from Balancer (0% fee)
Swap : USDC → ETH → USDC, profit +10 USDC
Repay : Approve Vault for 1000 USDC, return
Expected: Pool balance unchanged. Caller: +10 USDC. 1 TX.
Fail sim: Short repay (999 USDC) → entire TX reverts. Pool sees no loan.
