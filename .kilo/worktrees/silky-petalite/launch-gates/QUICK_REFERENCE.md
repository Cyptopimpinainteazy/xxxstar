## 🔍 Quick Audit Cycle (30 min per round)

1. **Generate packs** (5 min)
   ```bash
   ./repomix-mainnet-pack.sh
   ```

2. **Run audit** (10-20 min each, can do in parallel)
   - Copy `packs/full-repo-*.md` → Paste to Claude
   - Paste `prompts/01-wiring-audit.md`
   - Read results, save to `reports/01-wiring-audit-report.md`

3. **Track scores** (5 min)
   - Update MAINNET_READINESS.json with scores
   - Identify P0 blockers

4. **Fix blockers** (1-8 hours each depending on blocker)
   - Implement fix
   - Add tests
   - Commit

5. **Re-audit** (repeat)

## 📋 The 5 Audits Explained

### 1️⃣ **Wiring Audit** (01-wiring-audit.md)
**Goal:** Is every module actually reachable?  
**Pack:** full-repo-*.md  
**Output:** Unwired modules list + score (0-100)  
**Finding:** Dead code, orphaned pallets, incomplete integrations  
**Time:** 5-15 min  

### 2️⃣ **Mainnet Launch Gate** (02-mainnet-launch-gate.md)
**Goal:** 0-100 readiness score with exact blockers  
**Pack:** full-repo-*.md  
**Output:** Scorecard by category (consensus, runtime, bridge, storage), P0/P1/P2 blockers, time to fix  
**Finding:** Which systems need work, which are good  
**Time:** 10-20 min  

### 3️⃣ **Bridge Safety Audit** (03-bridge-safety-audit.md)
**Goal:** Find all cross-VM attack paths  
**Pack:** bridge-atomic-*.md  
**Output:** Threat model (replay, reorg, timeout, collusion), missing tests, score  
**Finding:** Specific exploit scenarios + how to block them  
**Time:** 20-30 min  

### 4️⃣ **Invariant Hunter** (04-invariant-hunter.md)
**Goal:** Every critical property is enforced + tested  
**Pack:** full-repo-*.md  
**Output:** Table of invariants (supply, atomicity, nonce, etc.) + enforcement/test/monitor status  
**Finding:** Missing enforcement, untested invariants, migration risks  
**Time:** 15-25 min  

### 5️⃣ **Test Gap Audit** (05-test-gap-audit.md)
**Goal:** All critical scenarios have tests  
**Pack:** tests-*.md  
**Output:** Coverage by area (atomicity, replay, finality, timeout, overflow, etc.), missing tests list  
**Finding:** Edge cases without tests + exact test names needed  
**Time:** 10-20 min
