# Frontend shell mock data

`frontendShellContractData.js` is not hand-maintained.
It imports `.launchops/frontend_route_allowlist.json` directly.

That file is generated by the LaunchOps scanner. Before running
`npm run build` in any environment that does not have `.launchops/`
checked out, run:

```
cargo run -p launchops -- scan
```

from the repo root. CI already does this in `.github/workflows/ci.yml`
and `.github/workflows/mainnet-gating.yml` before any LaunchOps consumer
step.

When the generated contract changes intentionally, update
`X3_LAUNCHOPS_CONTRACT_REVIEW_NOTE.md`; the
`scripts/ci/verify_launchops_contract_review.sh` gate enforces that.
