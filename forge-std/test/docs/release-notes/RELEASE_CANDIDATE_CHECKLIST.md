# X3 Release Candidate Checklist

## Purpose
Provide a repeatable, artifact-backed release candidate (RC) rehearsal checklist for the minimal mainnet hardening path.

## Current Reality
- The RC flow is gate-driven and depends on proof, parity, replay, determinism, and status-doc truth checks.
- This checklist does not mark release readiness by itself; it documents required evidence and operator sign-off steps.

## Verified
- CI gates exist for receipt schema checks, claim registry status, EVM or SVM parity, replay or failure-path checks, GPU or CPU determinism benchmark checks, and status-doc truth lint.
- A dedicated rehearsal workflow exists in .github/workflows/release-candidate-rehearsal.yml.

## Gaps / Risks
- This checklist cannot prove operational readiness unless artifacts are generated by an actual rehearsal run.
- Any skipped gate invalidates RC evidence integrity.

## Release Impact
- Blocks candidate publication if any required gate or sign-off item is missing.

## Next Required Work
- Run the release-candidate rehearsal workflow against the intended commit/tag.
- Publish evidence artifacts using the pack spec and collect sign-off.

## Checklist

### 1) Candidate Pinning
- [ ] Candidate tag or commit SHA is explicitly pinned.
- [ ] Candidate metadata includes operator, run date, and environment.

### 2) Required Technical Gates
- [ ] Receipt schema gate passed (`scripts/proof/verify-receipts.sh`).
- [ ] Claim registry status gate passed (`scripts/proof/check-claim-status.sh`).
- [ ] EVM or SVM parity vectors gate passed (`parity-core parity_vectors`).
- [ ] Replay or failure-path gate passed (`scripts/test-minimal-stack-replay-failure.sh`).
- [ ] GPU or CPU determinism benchmark gate passed (`scripts/test-gpu-cpu-determinism-benchmark-gate.sh`).
- [ ] Status-doc truth lint gate passed (`scripts/proof/lint-status-doc-truth.sh`).

### 3) Evidence Bundle Completeness
- [ ] Gate command outputs saved as text artifacts.
- [ ] Commit SHA, branch, and workflow run URL captured.
- [ ] Candidate evidence pack structure matches spec.

### 4) Sign-Off
- [ ] Engineering sign-off recorded.
- [ ] Security sign-off recorded.
- [ ] Operations sign-off recorded.
- [ ] Product or release-owner sign-off recorded.

### 5) Exit Criteria
- [ ] All required gates passed in one rehearsal run.
- [ ] Evidence pack published and immutable.
- [ ] Sign-off template completed with no unresolved blockers.
