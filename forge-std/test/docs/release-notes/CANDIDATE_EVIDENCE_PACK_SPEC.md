# Candidate Evidence Pack Specification

## Purpose
Define the minimum evidence artifact contract for an X3 release candidate rehearsal and sign-off cycle.

## Scope
Applies to RC rehearsal outputs generated by .github/workflows/release-candidate-rehearsal.yml.

## Required Directory Layout

```text
candidate-evidence/
  metadata/
    run.json
  gates/
    receipt-schema.txt
    claim-status.txt
    parity-vectors.txt
    replay-failure.txt
    gpu-cpu-determinism.txt
    status-doc-truth-lint.txt
  signoff/
    candidate-signoff.md
```

## Required Metadata (`metadata/run.json`)
- `candidate_ref`: tag or commit SHA used for rehearsal
- `workflow_run_id`: CI workflow run id
- `workflow_url`: CI run URL
- `commit_sha`: tested commit
- `timestamp_utc`: generation timestamp
- `operator`: initiating user or automation identity

## Gate Artifact Rules
- Each required gate emits one plain-text output file.
- File names must match the layout exactly.
- Non-zero exit from any gate means candidate evidence is incomplete.

## Sign-Off Artifact Rules
- `signoff/candidate-signoff.md` must follow the sign-off template.
- Every sign-off role must be either `APPROVED`, `BLOCKED`, or `NOT REVIEWED`.
- `BLOCKED` requires a blocker id and remediation note.

## Rejection Conditions
- Missing metadata keys.
- Missing any required gate output file.
- Modified file names or path drift from the spec.
- Any unresolved `BLOCKED` sign-off.
